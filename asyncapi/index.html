CyberCity Event Bus 0.0.3

Dieser Service kümmert sich um den Kommunikationsweg der einzelnen Microservices. Umgesetzt wird diese Funktionalität mit RabbitMQ. Ein Event wird immer von einem Event-Wrapper umschlossen. Im Nachfolgenden werden alle Events ohne diesen Wrapper aufgelistet. Die Beispiele enthalten jedoch immer den Wrapper. Außdem wird der Event-Wrapper einzeln und mit Beschreibung in dem Abschnitt "Messages" weiter unten auf der Seite als "#1" aufgelistet. Nur Events, die von dem Wrapper umschlossen werden, dürfen in den Event Bus geschrieben werden.

    Dieses Event wird gesendet, wenn eine Eilmeldung erstellt wurde, die in den Microsevices angezeigt werden soll.

    Accepts the following message:

    breaking_news_created
    object
    message
    string

    Inhalt der Eilmeldung.

    Additional properties are allowed.

    Examples

  "event_type": "breaking_news_created",

  "event_time": "2013-05-13T12:58:26Z",
  "content": {
    "message": "Das ist eine Eilmeldung"

  • SUB error_event

    Dieses Event wird als Antwort gesendet, wenn bei der Validierung eines Events ein Fehler aufgetreten ist. Dieses Event wird über eine spezielle Exchange gesendet, die extra für Error-Events erstellt worden ist.

    Accepts the following message:

    error_event
    Payload
    any

    Dieses Event beinhaltet eine Kopie des ursprünglichen fehlerhaften Events inklusive Event-Wrapper.

    Examples

  "event_origin": "microservice.krankenhaus",

  "content": {

    "event_type": "breaking_news_created",
    "event_origin": "microservice.umwelt",
    "event_time": "2013-05-13T12:58:26Z",
    "content": {
      "messaggge": 57

  • SUB air_quality_warning_issued

    Dieses Event wird gesendet, wenn eine Warnung zur Luftverschmutzung abgegeben wurde.

    Accepts the following message:

    air_quality_warning_issued
    object
    message
    string

    Die Nachricht, die bei der Warnung angezeigt wird.

    pollution_type
    string

    Enthält die Art der Luftverschmutzung und wird von dem sendenden Microservice bestimmt. Diese dient hauptsächlich zur programmatischen bestimmung der Art der Luftverschmutzung. Beispiel: Pollen oder Feinstaub.

    pollution_level
    string

    Grad der Luftverschmutzung. Die Einheit und der Wertebereich wird von dem sendenden Microservice bestimmt und ist abhängig von der Art der Luftverschmutzung. Die Einheit sollte auch in diesem Feld enthalten sein.

    Additional properties are allowed.

    Examples

    {
  "event_id": "AUT-70539",
  "event_type": "air_quality_warning_issued",
  "event_origin": "microservice.umwelt",
  "event_time": "2013-05-13T12:58:26Z",
  "content": {
    "message": "Achtung, es liegt viel Feinstaub in der Luft!",
    "pollution_type": "Feinstaub",
    "pollution_level": "20 µg/m³"
  }
}

Messages

  • #1event_wrapper
    object
    event_id
    string

    Enthält die ID des Events. Jedes Event muss eine einzigartige ID haben. Um zu gewährleisten, dass jedes Event-ID auch über alle Microservices einzig ist, sollte jede ID einen Präfix haben, der für jeden Microservice einzigartig ist, wie bspw. ein Abkürzung des Namens.

    event_type
    string

    Enthält die Art des Events. Dieses Feld muss den Namen eines Events enthalten und kann von empfangenden Microservices verwendet werden, um die Art des Events zu erkennen.

    event_origin
    string
    format: <typ>.<name>

    Enthält den Absender des Events. Dieses Feld muss den Namen eines Microservices enthalten (z. B. "microservice.krankenhaus") und kann von empfangenden Microservices verwendet werden, um den Absender des Events zu erkennen.

    event_time
    string
    format: ISO 8601 date and time

    Zeitpunkt der Eilmeldung, an dem diese ursprünglich erstellt worden ist. (Falls verspätete Zustellung.)

    object

    Beinhaltet das eigentliche Event.

    Additional properties are allowed.
